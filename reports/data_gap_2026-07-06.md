# 資料缺口觀察:外資持股(TaiwanStockShareholding)2026-07-06

## 現象

- `holding` 表(外資持股)在 2026-07-06 只覆蓋 79/98 檔;同一天 price/inst/margin/sbl
  這 4 個原始表都是 98/98 全滿。
- 對照近兩週(06-18 ~ 07-03),`holding` 每天都是 99/99(近乎全滿);2026-07-06 是
  這段期間唯一一次掉到 79。
- 2026-07-07 用 `fetch_daily.py --stocks <19檔> --datasets TaiwanStockShareholding
  --start 2026-07-06 --end 2026-07-06` 定向重抓,仍是 0 rows——排除「抓取時間點太早,
  晚點再抓就有」的可能性。

## 排查

- 07-06 那次 GitHub Actions run(28803288114,BOM bug 修復後的驗證 run)完整 log 裡,
  這 19 檔在任何 dataset 上都沒有出現 402 額度輪替、也沒有任何 `! ... 失敗` 例外訊息——
  代表 API 呼叫本身成功(HTTP 200),只是 FinMind 回傳的資料裡沒有這 19 檔 07-06
  這天的列。
- 判斷:較可能是 FinMind/TWSE 對這批股票的外資持股揭露有發布延遲,不是我們抓取
  邏輯的 bug(其餘 4 個 dataset、其餘 79 檔都正常)。

## 影響範圍

`score.py` 的 `rank_scores()` 對 None 值本來就是「不參與排名、給 0」,所以這 19 檔
2026-07-06 的③外資分數會顯示中性(0),不會報錯、也不會拖累同族群其他檔的排名——
影響有界,非結構性風險。

## 後續:2026-07-07 追蹤

- 07-07 定向重抓 FinMind 仍是 0 rows——連續兩天缺值,確認不是單純發布延遲,
  排除「等一天自癒」。
- 改查 TWSE MI_QFIIS(上市)/TPEx tpex_3insti_qfii(上櫃)這兩個交易所官方
  opendata,兩者皆免token、當天(07-06)就有這 19 檔的資料——證實是 FinMind
  轉載延遲,不是資料源真的沒發布。
- 已在 `fetch_daily.py` 加上 `backfill_holding_from_exchange()`:FinMind
  holding 缺值時自動查 TWSE/TPEx 補回,只補缺不覆寫既有資料,失敗不擋主管線
  (與 `fetch_tdcc.py` 同一原則)。用這 19 檔實測成功,重跑 score.py 後
  ③外資分數已從中性(0)更新為真實值。**結案**,自動化以後遇到同類缺值會
  自行補上,不需再手動追蹤。
